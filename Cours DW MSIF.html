Partie 2, XSLT

L'objectif de cet exercice est de créer des feuilles de styles XSL. Pour commencer, vous pouvez utiliser le fichier : cadre.xsl (présent dans l’archive du sujet) :

<?xml version="1.0" encoding="utf-8"?>

<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

  <xsl:output method="html"/>

  <xsl:template match="/">

    <html>

      <head>

        <title>XSLT Excercises</title>

      </head>

       <body bgcolor="beige">

        <H1>Company Information for Infoteria</H1>

        <H2>Employee Listing</H2>

        <table border="1">

          <tr>

            <th>Name</th>

            <th>Position</th>

            <th>E-mail</th>

          </tr>

        </table>

        <p>Total Employees:</p>

      </body>

    </html>

  </xsl:template>

</xsl:stylesheet>

 

Vous pouvez tester votre feuille de style avec EditX ou Firefox.

Pour cela:

  1. Ajoutez la ligne : <?xml-stylesheet type="text/xsl" href="cadre.xsl"?> au fichier Employees.xml juste après la première ligne.
  2. Pour permettre à Firefox de chercher la feuille de style sur le disque local, modifiez un paramètre du navigateur: allez sur la page : about:config , cherchez l'option : security.fileuri.strict_origin_policy, mettez sa valeur à faux.
  3. Ouvrez simplement le fichier Employees.xml avec Firefox.

Testez la systèmatiquement en ouvrant le fichier xml avec Firefox pour voir le résultat! Si il ya erreur, EditX est plus locace sur sa raison que Firefox !

Partie 3 Ajax

Utilisez les outils de développement de Firefox (bouton F12 sous Windows/Linux ) pour inspecter les éléments, consultez les messages d'erreur dans l'onglet : Console, exécutez pas à pas le javascript dans l'onglet Débogueur. Regardez le tutoriel officiel Mozilla pour vous familiariser avec ce processus.

Utilisez un éditeur de texte, par exemple Notepad++, Eclipse ou Brackets.io (portable si nécessaire) pour modifier le code source des pages.

Vous trouverez des informations complémentaires sur le site de w3schools.com sur les méthodes DOM en général et sur les méthodes DOM spécifique html

Dans le répertoire ajax du sujet, vous trouvez 3 fichiers html (PartieAjaxDOM_01.html, PartieAjaxDOM_02.html, PartieAjaxDOM_03.html).

  1. Ouvrez le fichier PartieAjaxDOM_01.html avec Firefox et avec votre éditeur de code. Cette page contient un bouton qui remplace un texte sur la page par le contenu d’un champ de saisi. Regardez le code source dans votre éditeur, lisez les commentaires et exécuter pas à pas le javascript dans Firefox. Quel est l’identifiant (valeur de l’attribut id) de l’élément qui est remplacé par le contenu du champ de saisi dans cet exemple ?
  2. Ajouter un second champ de saisi dans la page à côté du premier avec l’identifiant « myText2 ». Au clic sur le bouton afficher les contenus concaténés des deux boutons.
    • Ajouter un second paramètre à la fonction setNom avec le contenu du champ myText2
    • Chercher sur internet comment on concatène 2 chaines de caractères en javascript et afficher les 2 chaines de caractères sur la page (Si l’affichage ne se déroule pas comme prévu, allez dans l’onglet « Console » de la section de développement (F12) de Firefox et vérifiez qu’il n’y ait pas un message d’erreur, exécutez pas à pas éventuellement et vérifiez les valeurs de vos variables)
  3. Ouvrez le fichier PartieAjaxDOM_02.html avec Firefox et avec votre éditeur de code.  Cette page ajoute un second bouton qui lit le fichier Employee.xml, extrait les noms des employés et les affiche sur la page. Regardez le code source dans votre éditeur, lisez les commentaires et exécuter pas à pas le javascript dans Firefox. Quel est le nom de la variable dans laquelle on concatène les noms des employés ?
  4. Modifiez la fonction appelée au second bouton pour qu’elle affiche les prénoms et les noms de chaque employé.
  5. Affichez les employés dans une liste numérotée (balise <ol> et éléments <li>)
  6. Ouvrez le fichier PartieAjaxDOM_03.html avec Firefox et avec votre éditeur de code. Cette page charge un fichier XML et une feuille de style, transforme le fichier XML avec la feuille de style et affiche une partie du résultat sur la page html. Ouvrez les fichiers ajax.bib.xml et ajax.bib.xsl avec EditX, puis le fichier ajax.bib.xml avec Firefox. Examiner la page généré (bouton droit, puis Examine élément).
  7. Créez ensuite un Bouton 4 dans le fichier PartieAjaxDOM_03.html qui charge Employees.xml, le fichier xsl que vous avez créé pour la partie 2.2 (XSL), et qui affiche le résultat de la transformation du document XML par la feuille de style (le tableau des employés) en bas de la page html.
